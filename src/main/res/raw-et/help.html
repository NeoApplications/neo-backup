Backup modes: An app can be backed up with only its apk, only its data or both. An app not currently installed can only be restored if the apk is present.

Delete backup: Deletes the backup files for the chosen app.

Uninstall: Somewhat more aggressive than a normal uninstall. In addition to doing a normal uninstall via android commands, uninstalling from OAndBackupX deletes files the app might have left over in /data/app-lib/.

Enable / disable: Uses the android script `pm` to enable or disable an app. Disabling an app removes it from the normal user interface without uninstalling. This can be used for enabling or disabling an app for muliple users at a time (if the device has multiple users enabled). Users are identified with an id. 0 is the first user (owner).

Mitme kasutajaga nutiseadmed: Varundamine ja taastamine mitme kasutajaga seadmetel on osaliselt katseline, kuid peaks toimima. Sisse- ja väljalülitamine toimib ainult peale algset paigaldust (tingimata mitte OAndBackupxist) või rakenduste taastamist.

Süsteemivälised rakendused: Magiski moodulitena paigaldatud süsteemiväliseid rakendusi ei saa varundada ega taastada ning selle üritamine võib põhjustada kogu nutiseadme alustarkvara katkimineku, mille kordategemine eeldab operatsioonisüsteemi uut paigaldust.

Liikumine: Menüüribal on nupud „Seadistused“, „Avaleht“, „Pakkvarundus“, „Pakktaastamine“ ja „Varunduse ajastamine“ (selles järjekorras). Peale viimast varundust uuendatud rakenduste arv on kuvatud teavitusena nupul „Avaleht“.

Rakendused, mis kasutavad võtmerõngast: Androidis on olemas võtmerõngas (Keystore) privaatsete andmete salvestamiseks krüptituna. Rakendusi, mis kasutavad seda sisselogimis- või muude andmete turvaliseks hoidmiseks, ei saa töökindlalt taastada. Kaks näidet sellistest rakendustest on Signal ja Element (vana nimega Riot).
